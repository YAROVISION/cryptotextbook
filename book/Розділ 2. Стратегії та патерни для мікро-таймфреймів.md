# Розділ 2. Стратегії та патерни для мікро-таймфреймів

Скальпінг базується на пошуку та відпрацюванні чітких математичних та графічних закономірностей у поведінці ціни та ліквідності. Цей розділ описує основні напрямки угод, торгові сетапи для мікро-таймфреймів, роботу з кореляціями та математичний алгоритм розрахунку кута нахилу тренду.

---

## 1. Концепції ЛОНГ та ШОРТ у скальпінгу

Угоди в скальпінгу відкриваються в обох напрямках ринку:

- **Лонг (Long / Довга позиція)**: купівля активу за нижчою ціною з розрахунком на зростання його вартості та продаж дорожче. Вхід здійснюється перед великими щільностями покупця в стакані або під час імпульсного пробою рівнів опору.
- **Шорт (Short / Коротка позиція)**: продаж активу, взятого в борг у біржі за вищою ціною, з очікуванням падіння. Після зниження ціни актив викуповується дешевше, борг повертається біржі, а різниця становить чистий прибуток.

> [!TIP]
> **Ринкова аксіома**: «Ринки зростають по сходах, а падають на ліфті». Панічні розпродажі відбуваються значно швидше та агресивніше за висхідні рухи. Тому для скальпера шорт-позиції є найшвидшими та найприбутковішими через миттєве вивільнення ринкової енергії.

---

## 2. Торгові стратегії для мікро-таймфреймів

Усі входи в ринок повинні здійснюватися системно, виключаючи угоди від нудьги чи імпульсивних рішень.

### Стратегія 1. Імпульсний пробій рівня (Impulse Breakout)

Мета стратегії — зловити момент пробиття сильного горизонтального рівня або нахилки, за якою накопичені стоп-лосси роздрібних трейдерів, та забрати швидкий імпульсний рух ціни.

- **Анатомія сетапу**:
  - Наявність сильного рівня (бажано від 3–4 торкань на графіку).
  - Плавне підтискання ціни до рівня (мінімуми підвищуються для лонгу або максимуми знижуються для шорту).
  - Формування щільної бази проторговки безпосередньо під рівнем.
  - Наявність великої лімітної щільності продавця/покупця на самому рівні в стакані.
- **Вхід в угоду**: здійснюється в момент початку активного викупу (роз'їдання) лімітної щільності на рівні. У стрічці угод мають летіти лавиноподібні ринкові принти в бік пробою, а в стакані з'являтися обсяги підтримки, що виштовхують ціну.
- **Вихід з угоди**: закриття позиції лімітними ордерами на першому різкому імпульсі (зазвичай протягом кількох секунд), викликаному спрацюванням стоп-лоссів шортистів або лонгістів. Намагання пересидіти рух після згасання імпульсу призводить до вкату ціни назад.
- **Ризики**: хибні пробої (заколювання рівня без подальшого імпульсу) та проковзування ордерів (slippage) на високій волатильності.

### Стратегія 2. Відскок від щільності (Mean Reversion)

Контртрендова стратегія, заснована на відскоку ціни від великої лімітної заявки в стакані, яка значно перевищує стандартний середній об'єм інструмента.

- **Вхід в угоду**: виставлення лімітного ордера на купівлю чи продаж на 1–2 кроки ціни безпосередньо перед щільністю (**фронтранінг**).
- **Вихід з угоди**: фіксація прибутку на першому технічному відскоку ціни від щільності (зазвичай 0.2%–0.5%). Якщо щільність починають швидко розбирати (викуповувати великими ринковими принтами в стрічці), здійснюється негайний вихід по ринку, не чекаючи стоп-лоссу.
- **Захисний ордер**: стоп-лосс виставляється строго за саму щільність. Ризик мінімальний (кілька тіків), що забезпечує чудове математичне очікування угоди.

### Стратегія 3. Торгівля заколів та ловля ножів

- **Закол рівня (Хибний пробій)** — це ситуація, коли ціна на короткий момент перетинає сильний рівень опору чи підтримки, активує перші стоп-лосси та відкладені ордери, але не отримує підтримки обсягів, швидко розвертається й повертається під рівень. На графіку утворюється довгий хвіст (тінь свічки).
  - *Тактика торгівлі заколу*: скальпер дає ціні пробити рівень та активувати ліквідність. Коли в стрічці угод спостерігається явне сповільнення швидкості принтів, а в кластері на кінці тіні свічки фіксується Point of Control (POC), трейдер відкриває позицію у протилежний бік (контртренд). Стоп-лосс виставляється за кінчик утвореної тіні (голки).
- **Ловля ножів (Knife Catching)** — вхід у позицію під час стрімкого вертикального падіння ціни (проливу) з метою взяти швидкий технічний відскок.
  - *Правила безпеки при ловлі ножів*: категорично заборонено купувати падаючий актив без підтвердження. Необхідно дочекатися зупинки руху в стрічці угод, появи великої лімітної підтримки в стакані (біда) та наторговки максимального об'єму внизу свічки в кластерах (POC утворює "замок" ліквідності).

### Стратегія 4. Збір спреду на неліквіді

Використовується на малоліквідних активах чи у моменти після лістингів, де різниця між найкращим Bid та Ask є значною у відсотках.

- **Вхід в угоду**: скальпер виступає у ролі локального маркетмейкера, одночасно виставляючи лімітний ордер на купівлю на 1 крок ціни вище найкращого Bid та лімітний ордер на продаж на 1 крок ціни нижче найкращого Ask.
- **Вихід з угоди**: виконання обох ордерів ринковими заявками інших учасників. Різниця (спред) залишається трейдеру як прибуток.
- **Правило безпеки**: торгувати тільки лімітними ордерами для сплати найменшої комісії Maker (див. `[[Розділ 4. Специфіка крипторинку]]`) та невеликим об'ємом через ризик різкого виходу ціни зі спреду.

## 3. Кореляції та ринкові поводирі (Guides)

Рух криптовалют рідко буває повністю ізольованим. Більшість активів корелюють між собою та підпорядковуються загальним тенденціям ринку.

- **Ринкові поводирі (Guides)** — активи-лідери, які задають вектор руху всьому ринку. Головним поводирем є **Bitcoin (BTC)**, другорядним — **Ethereum (ETH)**.
- **Робота з поводирями**:
  - При торгівлі будь-якого альткоїна скальпер зобов'язаний тримати на екрані стакан та графік BTC.
  - Якщо на альткоїні формується ідеальний лонг-сетап на пробій рівня опору, але в цей момент BTC починає стрімко падати — лонг-сетап з високою ймовірністю буде зламано (відбудеться закол). Вхід скасовується.
- **Сила та слабкість активу**:
  - *Сильна монета*: BTC падає, а обраний альткоїн стоїть на місці або навіть росте. Це потенційний кандидат на ЛОНГ при першій же стабілізації BTC.
  - *Слабка монета*: BTC росте, а альткоїн падає чи стоїть у боковику. Кандидат на ШОРТ при першій зупинці або падінні BTC.

## 4. Математична модель: Розрахунок кута нахилу тренду (OLS-регресія)

Для автоматизації аналізу швидкості (імпульсу) та напрямку тренду на мікрорівні скальпери розраховують вектор зміни ціни на основі **лінійної регресії (методу найменших квадратів, OLS)** за ковзним вікном з останніх $N$ тіків (типове значення $N = 25$).

### Побудова системи координат

- **Вісь X (Час/Тіки)**: індекси тіків від $0$ до $N-1$ ($N = 25$). Початок координат ($0$) — це точка 25 тіків тому, поточний тік має координату $24$.
- **Вісь Y (Нормалізована ціна)**: для усунення проблеми зміни вартості самого активу, зміна ціни вимірюється в мінімальних кроках ціни (тіках/пунктах):
  $$Y_i = \frac{\text{Ціна}_i - \text{Ціна}_{N \text{ тіків тому}}}{\text{tick\_size}}$$

### Математичний розрахунок

Коефіцієнт нахилу ($slope$) лінії регресії, яка найкраще апроксимує тренд, розраховується за формулою:
$$slope = \frac{N \sum_{i=0}^{N-1} (x_i y_i) - \sum_{i=0}^{N-1} x_i \sum_{i=0}^{N-1} y_i}{N \sum_{i=0}^{N-1} x_i^2 - \left(\sum_{i=0}^{N-1} x_i\right)^2}$$

Отриманий коефіцієнт $slope$ є тангенсом кута нахилу. Кут у градусах ($\alpha$) знаходять за допомогою арктангенсу:
$$\alpha = \arctan(slope) \times \frac{180}{\pi}$$

Діапазон значень кута $\alpha$ становить від $-90^\circ$ до $+90^\circ$:

- $\alpha > 0$ — тренд спрямований вгору (лонг-вектор).
- $\alpha < 0$ — тренд спрямований вниз (шорт-вектор).
- $\alpha = 0$ — повна відсутність спрямованого руху (флет).

### Сигнали ("Ватерлінія")

- $\alpha > +45^\circ$ — висхідний імпульс, сигнал на ЛОНГ.
- $\alpha < -45^\circ$ — спадний імпульс, сигнал на ШОРТ.

### Код реалізації OLS-регресії на Python

Нижче наведено оптимізований клас для обчислення кута нахилу лінії тренду в реальному часі за допомогою ковзного вікна:

```python
import math
from collections import deque

class ScalpingTrendVector:
    def __init__(self, window_size=25, tick_size=0.1):
        self.window_size = window_size
        self.tick_size = tick_size
        self.prices = deque(maxlen=window_size)
        
        # Передобчислення статичних сум для осі X (індекси 0..24) для підвищення швидкодії
        self.x = list(range(window_size))
        self.sum_x = sum(self.x)
        self.sum_x_sq = sum(xi ** 2 for xi in self.x)
        self.denominator = window_size * self.sum_x_sq - (self.sum_x ** 2)

    def add_tick(self, price):
        self.prices.append(price)
        if len(self.prices) < self.window_size:
            return None  # Недостатньо даних для розрахунку
        
        # Нормалізація цін Y відносно стартової ціни поточного вікна
        initial_price = self.prices[0]
        y = [(p - initial_price) / self.tick_size for p in self.prices]
        
        # Обчислення сум
        sum_y = sum(y)
        sum_xy = sum(self.x[i] * y[i] for i in range(self.window_size))
        
        # Розрахунок slope за формулою OLS
        slope = (self.window_size * sum_xy - self.sum_x * sum_y) / self.denominator
        
        # Розрахунок кута в градусах
        angle_radians = math.atan(slope)
        angle_degrees = math.degrees(angle_radians)
        
        return angle_degrees

    def check_signal(self, angle):
        if angle is None:
            return "Очікування накопичення тіків..."
        if angle > 45.0:
            return "СИГНАЛ: ІМПУЛЬСНИЙ ЛОНГ (Кут: {:.2f}°)".format(angle)
        elif angle < -45.0:
            return "СИГНАЛ: ІМПУЛЬСНИЙ ШОРТ (Кут: {:.2f}°)".format(angle)
        return "Флет / Діапазон (Кут: {:.2f}°)".format(angle)
```
